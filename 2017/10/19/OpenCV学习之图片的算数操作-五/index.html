OpenCV学习之图片的算数操作(五)

发表于 2017-10-19 | 阅读次数:

图片的算数操作,包括图片的加法, 减法, 位操作等.

图片加法

我们可以用 cv2.add() 方法对图片进行加法操作, 也可以直接通过numpy的加法来进行操作, 例如res = img1 + img2, 两张图片的通道数,长宽,数据类型必须相同, 或者img2直接为一个常数, 也开始可以的.
注意: OpenCV的加法和Numpy的加法是不同的, OpenCV的加法是一种饱和操作, 假设参与求和运算的两个数的数据类型都为uint8,uint8的范围是[0, 255], 正好对应一般像素值的取值范围, 如果和超过了255, 则取255. 而Numpy的加法是一种模运算, 假设和为260, 基数为256, 此时进行取余,最终结果为4. 

x = np.uint8([250])
y = np.uint8([10])

print cv2.add(x, y)
print x+y

位操作

位操作包括与/或/非/异或, 其中异或只有当两者都不同时为真, 这些操作对于我们抽取出图片的某一个区域而言非常有用, 前面我们操作过图片的某一个区域, 不过那只能是矩形区域,对于非矩形区域,我们通过切片的方式就不好访问了.

如果我想把opencv的logo放到另外一张图片上边, 如果我直接对两张图片进行求和, 这样会改变图片的颜色, 如果我进行混合,给一个权重系数, 这时会有一种透明的效果,但是我不想要这个透明效果,如果logo是一个矩形区域,我们可以按照前面的方式, 直接操作.前面的方式都不行,这时,opencv的位操作隆重登场了!!!

我们的大体思路是,将opencv的logo部分从opencv的logo里边抽取出来,即形成一个类似png格式的图片,背景透明,然后将想要放上目标图片上放置logo的部分抠空,前景和背景都处理好之后,直接想加,就得到我们想要的效果了. 

# Load two images
img1 = cv2.imread('messi5.jpg')
img2 = cv2.imread('opencv_logo.png')

# I want to put logo on top-left corner, So I create a ROI
rows,cols,channels = img2.shape
roi = img1[0:rows, 0:cols ]

# Now create a mask of logo and create its inverse mask also
img2gray = cv2.cvtColor(img2,cv2.COLOR_BGR2GRAY)
ret, mask = cv2.threshold(img2gray, 10, 255, cv2.THRESH_BINARY)
mask_inv = cv2.bitwise_not(mask)

# Now black-out the area of logo in ROI
img1_bg = cv2.bitwise_and(roi,roi,mask = mask_inv)

# Take only region of logo from logo image.
img2_fg = cv2.bitwise_and(img2,img2,mask = mask)

# Put logo in ROI and modify the main image
dst = cv2.add(img1_bg,img2_fg)
img1[0:rows, 0:cols ] = dst

cv2.imshow('res',img1)
cv2.waitKey(0)
cv2.destroyAllWindows()

cv2.threshold()函数的作用是二值化图像, 第一个参数是原图,参与二值画的图像,必须是一个灰度图, 第二个参数是一个阈值, 高于这个值,将其置为第三个参数的值,否则置为0, 第四个参数是二值化的模式, 前面几个参数的描述是基于cv2.THRESH_BINARY这个模式的.

cv2.bitwise_not()的作用是进行取反.

cv2.bitwise_and()的作用是进行按位与, 这是一个双目操作符, 必须传入两张图片,作为与操作的对象, 第三个参数mask的作用是, 设定图片参与与操作的区域, 具体做法是,两张图片分别先与mask求与, 两张图片再进行与运算.
