北邮人论坛搜索引擎开发教程

发表于 2017-11-11 | 阅读次数:

之前看别人在byr论坛上发帖,说自己开发了一个论坛搜索引擎,顿时引来一大波围观,不仅因为感觉这个东西很高端,更主要的原因是byr论坛真的需要一个搜索引擎,来方便童鞋们进行相关搜索,当时感觉这个东西真的好神奇啊.经过相关学习,最近也实现了一个byr论坛搜索引擎,在此记录学习的点滴,也给感兴趣的童鞋提供一些借鉴.

直观想来,搜索引擎也就是根据用户输入的关键字, 从数据库里边取出与关键字相关的内容, 并呈现在网页上的过程,如果不考虑效率的话,完全可以写个for循环, 拿着关键字和数据库里边的内容逐条对比,符合条件就取出来.这种方式简单粗暴, 但是当数据库里边的数据量很大的时候,用这种方式就会耗费很多时间,效率很低,因此就出现了很多关于提升搜索效率的算法,本文不介绍这些算法,我们的搜索引擎也不自己从底层实现, 而是使用一个集搜索之大成的插件elasticsearch, 这个插件驰名海内外,很多大公司都在使用,比如微软/惠普…

经过以上介绍,byr论坛搜索引擎的实现由以下部分组成:
　　1.搜索的数据,从哪搜索.本文使用scrapy从北邮人论坛上获取数据,并存入数据库.
　　2.搜索的方法,如何搜索.本文使用elasticsearch进行搜索.
　　3.web页面,负责显示搜索结果.本文使用django进行web搭建.　　

写到这本该有个高大上的框架图,画图有点麻烦,以上解析就已经简单明了了…下边分别就以上3部分进行讲解.

scrapy爬取北邮人论坛

step1 模拟登录

北邮人论坛必须先登录才能访问,因此首先是模拟登陆.首先打开北邮人论坛的登录页面https://bbs.byr.cn/index, 打开调试模式,分别输入用户名和一个错误的密码(之所以输入错误的密码是为了防止登陆成功页面跳转,页面一旦跳转了不利于进行调试),如下图所示:

如图中红框标记,北邮人论坛登录是向 https://bbs.byr.cn/user/ajax_login.json 发送了一个请求,方法是POST, 参数在Form Data里边,分别有 id / passwd / mode / CookieDate, 我们通过分析出的这些信息,由python向服务器将这些参数POST过去,就实现了登录,scrapy代码如下

至此我们完成了北邮人论坛的登录,登陆成功之后,北邮人论坛服务器会在给我们生成一个cookie文件,这个cookie文件类似于一个凭证,每次请求的时候都把这个cookie带过去, 服务器就知道这个家伙之前登陆过,不需要再登陆了,会把登陆之后的页面返回给我们,这样,我们就可以爬取北邮人论坛的内容了.

step2 分析网站结构,制定爬取策略

登陆之后,进入北邮人论坛首页,经过分析得知,北邮人论坛总共有10大板块,每个板块下面又有小的板块,比如谈天说地/缘来有你…要爬取北邮人论坛的所有文章,也就是把每一个子版块的文章都爬下来,为此我们遍历10大板块,分别取出他们各自的子版块的地址,再根据子版块的地址进入该子版块,将该子版块的所有文章都爬下来,这个过程是一个广度优先算法的过程,当然也可以设计成深度优先,都是可以的.

step3 设计数据库

可以根据自己的需求,设计不同的数据库,此处获取每一篇帖子的所在列表页的url, 帖子的url, 帖子的标题,帖子的创建时间,作者的id, 回复数目, 最后一次回复时间, 最后一次回复作者的id, 爬取时间,为方便,不获取帖子回复的具体内容.
其中有一项”url_object_id”,这是对文章url进行md5编码之后的结果,这样会使不同长度的url变成等长,且是唯一的,用作数据表主键.数据表设计如下图:

此处使用的是mysql数据库,如果单纯进行数据存储,如上设计数据库即可.

但我们要设计一个北邮人搜索引擎,就不能使用mysql数据库了.由于我们使用了elasticsearch这个搜索插件,这个插件自带一个数据库,我们要使用它自带的这个数据库,elasticsearch自带的数据库和mysql数据库不同,前者是no-sql数据库(意为:not only sql), 后者是关系型数据库. 基本思路是在爬取数据的时候,将数据存入elasticsearch自带的数据库 ,并对数据进行一定处理(加快搜索效率), 搜索的时候elasticsearch从它这个自带的数据库里进行搜索.

此处涉及到elasticsearch了,elasticsearch的数据库该如何设置?在elasticsearch处进行讲解.

step4 解析页面,数据入库

根据前边获得的每一个板块的url,对每一个板块进行爬取.基本思路是,解析帖子列表页,获取每一篇帖子的url,根据帖子url向服务器发起请求,从服务器的返回的内容里边,抽取我们想要获取的信息,比如文章标题,作者id等,然后再讲获得的这些信息存入elasticsearch的数据库.

elasticsearch的使用

搜索大致思路:每爬取一个字段,将这个字段放入elasticsearch数据库的时候,我们可以定义一个分词器对每一个字段进行分词,例如:我爱祖国,分词其就会将这句话分成:我/爱/祖国,这样就会生成的很多的词语, 我们建立一个词语索引表, 每一个索引后边都跟着带有这个词语的文章/标题,这样,每次搜索的时候,elasticsearch对我们的搜索关键词也进行分词,直接拿着这些分词去索引表里边取出该词语后边的所有相关的文章,这就是所谓的倒排索引,效率很高,也是elasticsearch搜索基本原理.

step1 elasticsearch-rtf的安装

由于我们要对中文进行搜索, 但是原生的elasticsearch是服务于英文搜索的,因此为了使用elasticsearch,我们要安装很多插件, 所幸,前人已经帮我们把所有的一些都做好了,将所有的插件和elasticsearch集成在了一起,可以很好支持中文的搜索, 这也就是elasticsearch-rtf.

安装方式,进入这个网站,单击此处,按照里边的方式进行安装即可.注意elasticsearch依赖java环境,因此要安装JDK, 且版本是8.0以上,注意此处elasticsearch-rtf中的elasticsearch的版本号, 写该教程时, 版本号为5.1.1(因为elasticsearch-rtf是集成了elasticsearch,因此elasticsearch-rtf中的elasticsearch的版本号不一定与elasticsearch官方同步,一般要低于官方版本),后边要用到.

JDK8.0安装好之后,进入从github上克隆下来的elasticsearch-rtf的bin目录,直接运行里边的elasticsearch即可

step2 elasticsearch-head安装

elasticsearch-head就是对elasticsearch数据库的可视化,使我们可以在界面上进行对数据的增删该查,而不是进行命令操作,相当于MySQL的navicat(navicate是对MySQL数据库进行可视化的工具). elasticsearch-head使用点击此处, elasticsearch-head的安装需要使用npm管理包,因此需要安装npm,npm是nodejs的包管理工具,因此安装上nodejs,npm就装上了,因此进行nodejs安装,从官网上选择与自己系统一致的的nodejs安装即可.

安装完之后,由于npm使用的是国外的源,速度很慢, 有一个淘宝镜像源,是国内的源,速度会很快,配置使用淘宝镜像源,进入这个网址进行配置 点击此处, 配置完成之后, 所有的npm命令用cnpm命令代替,其他与原来保持一致,速度会快很多,例如,原来:

step5 elasticsearch数据表设计

至此,有关elasticsearch的所有工具安装完毕,运行如下代码:

# -*- coding: utf-8 -*-

from elasticsearch_dsl import DocType, Date, Integer, Keyword, Text
from elasticsearch_dsl.connections import connections
from elasticsearch_dsl import Completion
from elasticsearch_dsl.analysis import CustomAnalyzer as _CustomAnalyzer

# 与elasticsearch数据库连接
connections.create_connection(hosts=['localhost'])


class CustomAnalyzer(_CustomAnalyzer):
    def get_analysis_definition(self):
        return {}

# 定义分析器,lowercase表示进行大小写转换
ik_analyzer = CustomAnalyzer('ik_max_word',filter=['lowercase'])


# 这个类定义elasticsearch保存的一些字段,即fields
class ArticleType(DocType):
    # 如果我们想有只能提示,就要添加一个Completion类型的field,定义这个field和其他field方式是一样,只不过
    # 源码的问题,我们这个地方在传入analyzer时,要重新定义一个CustomAnalyzer,并把ik_max_word传入,如上
    suggest = Completion(analyzer=ik_analyzer)
    title = Text(analyzer="ik_max_word")
    create_date = Date()
    url = Keyword()
    url_object_id = Keyword()
    front_image_url = Keyword()
    front_image_path = Keyword()
    praise_nums = Integer()
    comment_nums = Integer()
    fav_nums = Integer()
    tags = Text(analyzer="ik_max_word")
    content = Text(analyzer="ik_max_word")

    # 此处指定index/type
    class Meta:
        index = 'jobbole'
        doc_type = 'article'

if __name__ == "__main__":
    ArticleType.init()

使用lasticsearch-dsl完成了对elasticsearch的连接,并设计了数据表,进行了字段映射(也就是字段的数据类型), 生成我们之前感兴趣的数据条目的数据表,诸如文章标题,作者id,从代码能看出一二来,此处不具体讲解,如有兴趣,参看官方文档

在第一部分爬取北邮人论坛过程中,我们的抽取到的数据就是放到这个数据库中,不放到mysql中.
